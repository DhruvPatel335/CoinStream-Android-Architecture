The `REPORT_FORMAT.md` file defines the standardized structure for reports generated by the **R8 Analyzer** tool. The report is designed to help developers identify why R8 might not be optimizing their application fully and suggests actionable steps to improve app size and performance.

The report format consists of the following six sections:

### 1. Configuration (Optional)
This section highlights environment-level improvements.
*   **AGP Version**: Recommends upgrading to the latest Android Gradle Plugin (e.g., 9.0).
*   **Full Mode**: Checks if R8 "Full Mode" is enabled. It suggests removing `android.enableR8.fullMode=false` from `gradle.properties` to allow more aggressive optimizations.

### 2. Global Disable Rules (Optional)
Identifies rules that globally disable R8 features.
*   **Action**: Recommends removing flags like `-dontobfuscate`, `-dontoptimize`, or `-dontshrink` which prevent R8 from performing its core functions.

### 3. Optimization Summary
Provides a high-level percentage-based overview of the codebase's "health" regarding R8:
*   **Optimization Score**: Percentage of code available for inlining and merging.
*   **Shrinking Score**: Percentage of code that R8 can actually remove if unused.
*   **Obfuscation Score**: Percentage of the codebase available for renaming.
*   *Note: Higher scores indicate a smaller, more optimized app.*

### 4. Keep Rules Evaluation
This is a granular analysis of specific `-keep` rules and their impact:
*   **Impact Metrics**: Shows exactly how many classes, fields, and methods are "frozen" by a specific rule.
*   **Kept Items**: Lists specific classes affected.
*   **Recommended Action**: Suggests whether to **Remove** the rule (e.g., if it's a duplicate from a library) or **Refine** it (e.g., using a more "surgical" rule instead of a wildcard).

### 5. Subsumed Keep Rules (Optional)
Identifies redundant rules that are already covered by broader rules.
*   **Action**: Recommends removing the redundant rule to keep the ProGuard configuration clean.

### 6. Historical Analysis Summary
If a previous report exists, this section compares the current scores against the previous ones to track improvements in Optimization, Obfuscation, and Shrinking percentages.

***

For more information on optimizing your build with R8, refer to the [Shrink, obfuscate, and optimize your app](https://developer.android.com/studio/build/shrink-code) guide on the Android Developers site.
