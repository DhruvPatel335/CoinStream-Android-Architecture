Based on the documentation provided, here is the guide for using the R8 Configuration Analyzer to identify impactful keep rules and evaluate your app's optimization scores.

### 1. Requirements
*   **R8 Version:** 9.3.7-dev or later.

### 2. Setup and Cleanup
Before running the analysis, ensure the output directory exists and is clear of data from previous runs:

```bash
# Create the directory
mkdir -p "$PWD/tmp/r8analysis"

# Remove intermediate files and move previous results to history
rm -f tmp/r8analysis/keepruleradius.json
rm -f tmp/r8analysis/*.pb

if [ -f tmp/r8analysis/analysis_result.txt ]; then
  cat tmp/r8analysis/analysis_result.txt > tmp/r8analysis/history.txt
  rm tmp/r8analysis/analysis_result.txt
fi
```

### 3. Generate the Configuration Report
Run your R8-enabled build (typically `assembleRelease`) while passing the `dumpkeepradiustodirectory` system property to point to your analysis folder:

```bash
./gradlew assembleRelease \
  -Dcom.android.tools.r8.dumpkeepradiustodirectory=$PWD/tmp/r8analysis
```

### 4. Process and Analyze Results
The analyzer produces Protobuf files that must be converted to JSON for analysis.

1.  **Prepare Python Bindings:** You must have a file named `keep_radius_pb2.py` in your directory. (The raw content for this file is provided in Step 10 of the reference document).
2.  **Convert to JSON:** Use a Python script to parse the `.pb` files generated by R8 into a readable JSON format (`keepruleradius.json`).
3.  **Run Analysis:** Run an analysis script on the JSON to calculate:
    *   **Optimization Score:** Percentage of items allowed to be optimized.
    *   **Obfuscation Score:** Percentage of items allowed to be obfuscated.
    *   **Shrinking Score:** Percentage of items allowed to be removed if unused.

### 5. Identifying Impactful Rules
The documentation provides a reporting script that calculates the "blast radius" of keep rules. It identifies:
*   **Top 5 Impactful Keep Rules:** The rules responsible for keeping the largest number of classes, fields, and methods.
*   **Subsumed Rules:** Rules that are redundant because their impact is already covered by another broader rule.

The output is saved to `tmp/r8analysis/analysis_result.txt` and includes scores like:
```text
Optimization Score: 85.00%
Obfuscation Score: 72.50%
Shrinking Score: 90.00%
```

### 6. Final Cleanup
Once you have recorded your results, you can remove the intermediate binary and JSON files:
```bash
rm tmp/r8analysis/keepruleradius.json
rm tmp/r8analysis/*.pb
```
